虚拟dom是什么？

1. vue2.x 才有虚拟dom

2. 本质就是一个js对象 => 有跨平台的特性

虚拟dom在vue中做什么？

1. vue的渲染过程(html, css, js)

template(vue) --> render() h() --> 虚拟dom --> 真实dom （第一次渲染） | | | | ul > li * 3 --> createElement(ul) --> js对象 --> 浏览器最终呈现 （ N 次渲染）

2. 将真实dom转化为虚拟dom (js对象) ==> 更新的时候做对比

虚拟dom是如何提升vue的渲染效率的？

1. 局部更新(节点数据)

2. 将直接操作dom的地方拿到两个js对象之中去作比较

虚拟dom生成的三个要素

1. 当前目标元素

2. 目标元素上的相关属性

3. 子节点

老的ul ==> js对象 | (布局更新 ==> 性能优化) ==> 两个li ==> 更新节点 | 新的ul ==> js对象
